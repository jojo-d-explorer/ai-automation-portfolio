**Dates:** January 26 - February 4, 2026

## What I Built

### Core Automation
- Multi-board job search (Greenhouse, Lever, Ashby)
- Resume-based scoring (1-100 scale, 70+ threshold)
- Automated deduplication across platforms
- CSV export with customizable fields

### Infrastructure
- Organized file structure (weekly results folders)
- Categorized prompt library (daily/weekly/for_others/experiments)
- GitHub portfolio with version control
- Reusable search templates

### Real-World Applications
- Ran 15+ job searches across multiple boards
- Found ~100+ jobs, ~30 scoring 70+
- Helped friend (Phil) with their search (60 results, 9 top matches)
- Tested searches for Manila, Bogota, Paris, DC, London

---

## Key Learnings

### Technical Insights
1. **Browser automation > direct scraping** - CoWork avoids rate limiting by acting like a human browser
2. **Shallow vs deep trade-off** - First page (fast) vs full pagination (thorough)
3. **Resume scoring is powerful** - Filters 100 jobs to 10 relevant ones automatically
4. **File organization matters** - Weekly folders prevent chaos as data accumulates
5. **Prompt structure = success** - Clear sections, explicit instructions, edge case handling

### Prompt Engineering Evolution
- Started: Vague instructions, missing edge cases
- Now: Structured prompts with customization checklists, 70-80% correct first try
- Learned: Be explicit about file paths, column order, deduplication logic

### Workflow Discoveries
- CoWork progress matrices are invaluable for monitoring searches
- Score rationales help prioritize applications (not just numbers)
- Comparing week-over-week results identifies truly new jobs
- Template library prevents reinventing prompts each time

---

## What Worked Really Well

1. **Resume-based scoring** - Game changer for prioritization
2. **Multi-board searches** - Cast wider net than single platform
3. **Helping others** (Phil) - Validated templates are reusable
4. **Organized file structure** - Easy to find past searches
5. **GitHub Desktop** - Makes version control accessible

---

## What Was Harder Than Expected

1. **File path navigation** - Spatial reasoning challenges made this tricky
2. **Prompt precision** - Small wording changes cause big behavior differences  
3. **Understanding CoWork's limits** - Learned it does shallow searches by default
4. **Git workflow** - Initial setup (iCloud issues, folder structure) was confusing
5. **Salary data gaps** - 50-70% of jobs don't list compensation

---

## Friction Points Still Present

1. **Scoring granularity** - Hard to understand why job X = 92 vs 80
2. **Manual customization** - Still updating dates/roles/locations in prompts each time
3. **CSV review workflow** - Opening/scanning files is manual
4. **No application tracking** - Found jobs, but not tracking which ones I applied to
5. **Funding status research** - Takes extra time, often unavailable
